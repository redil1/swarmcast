# SwarmCast Production Backlog

This backlog converts `STREAMING_PLATFORM_BLUEPRINT.md` into an end-to-end build plan for a production-ready zero-CDN streaming platform. Quality is the priority: every milestone has explicit acceptance criteria, observability, tests, and rollback expectations.

## Operating Principles

- Build in vertical slices that can be tested in isolation.
- Do not scale the fleet before proving the single-node pipeline and P2P offload.
- Treat sections 21 and 22 of the blueprint as canonical where earlier sections conflict.
- Keep the product playable from the Delivery Fleet before depending on P2P.
- Measure real offload ratio `rho`; never size delivery boxes from theoretical assumptions alone.
- Prefer boring, inspectable infrastructure over orchestration complexity until scale requires it.
- No third-party CDN paths are allowed in production. All fallback delivery must use owned Hetzner edge nodes.
- Runtime media base URL config rejects known third-party CDN provider hostnames.

## Definition Of Done

A backlog item is done only when all applicable points are true:

- Code is implemented, reviewed, and merged behind a feature flag where risk warrants it.
- Unit or integration tests cover the main behavior and at least one failure path.
- Metrics, logs, and alerts exist for production-impacting behavior.
- Secrets are not committed and are loaded from runtime configuration.
- Local or staging smoke tests pass with documented commands.
- Failure behavior is understood: retry, degrade, shed, or alert.
- Documentation or runbook updates are included when operators need them.

## Production Launch Gates

These gates must pass before public launch:

- Legal redistribution and peer relay rights are confirmed in writing.
- Delivery-Fleet-only playback works for every active channel.
- P2P playback works on real Android devices with hash verification.
- RLNC path from section 22 is active for production swarms.
- No app or server code references a third-party CDN endpoint.
- Edge cache repeat fetch shows `X-Cache: HIT`.
- Tracker restart does not break already-buffered playback; clients continue through Delivery Fleet.
- Poisoned peer is detected, scored down, and disconnected within two bad transfers.
- 200-peer WiFi-majority test reaches `rho >= 0.90` with stall rate `< 1%`.
- Dashboards show peers, active channels, edge egress, origin egress, `rho`, cache hit ratio, stall rate, startup latency, and buffer health.
- Autoscaling plan is tested in staging or manually documented for day-one operations.
